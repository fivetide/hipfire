# Quant quality eval — KLD-primary harness

Tracking issues #113 (uniform MQ-family quality) and #116 (Lloyd
quality). Plan: `docs/plans/issue-113-quant-quality-eval.md`
(rev-3.1).

This directory holds the eval harness: slice + scripts +
canary fixture + reference manifest + result tables.

## Layout

```
benchmarks/quality-baselines/
├── README.md                    # this file
├── slice/                       # frozen prompt bytes + tokenizer-parity inputs
│   ├── README.md
│   ├── make_slice.sh            # generator from wikitext-2 train
│   ├── slice.md5                # checksum tripwire
│   ├── wikitext2-1024s-2048ctx.txt   # generated by make_slice.sh
│   └── tokens.bin               # u32 token IDs (post-bridge or skipped)
├── harness/                     # the actual harness scripts + format readers
│   ├── README.md                # how-to-add-quant
│   ├── manifest.json            # SHA-pinned reference index
│   ├── kld_reduce.py            # bootstrap CI + result-table emitter
│   ├── kldref_format.py         # hipfire ref-format reader
│   ├── tokenizer_parity.py      # Step 1.5 tokenizer-parity check
│   ├── eval_gguf.sh             # llama.cpp wrapper (GGUF anchor track)
│   └── canary.md                # 11-seq fixture with expected KLDs
├── refs/                        # downloaded (gitignored) BF16 ref blobs
│   └── .gitignore
└── results/                     # output tables + plots
    └── README.md
```

## Workflow (overview)

1. **One-time** — generate the slice via `make_slice.sh`, run
   tokenizer-parity check, dump BF16 references on gfx1151 (via
   `build_kld_ref.rs`, lands in Step 2 of the plan), upload to
   `hipfire-models/hipfire-eval-refs`, fill manifest.

2. **Per quant variant** — run `eval_hipfire.rs` (or
   `eval_gguf.sh` for GGUF anchors) to compute per-sequence KLDs
   against the cached reference. Output: a small per-sequence-KLD
   file under `results/<date>/per-seq/<variant>-<arch>.bin`.

3. **Aggregate** — `kld_reduce.py` reads per-sequence-KLD files,
   bootstraps 95% CIs, emits the result table as markdown + the
   Pareto plot data.
